Detailed Description

template<typename T>
class cgs::ecs::ComponentStorage< T >

Sparse-set component storage.

Memory layout:

sparse_ [entity.id] -> dense index (or kInvalidIndex)
dense_ [index] -> component data
entities_[index] -> entity.id that owns dense_[index]
versions_[index] -> change counter for dense_[index]

All mutating operations bump a global version counter; the per-component version is set to globalVersion_ at the point of modification so that systems can efficiently detect stale data.
