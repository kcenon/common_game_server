Detailed Description

Manages the full plugin lifecycle: Load → Init → Active → Shutdown → Unload.

Supports two loading modes:

  • Dynamic: load shared libraries (.so/.dll) at runtime via LoadPlugin().
  • Static: register compile-time plugins via RegisterStaticPlugins().

The manager enforces a state machine for each plugin and resolves dependency ordering before initialization.

Initialize all loaded plugins in dependency order.

Resolves dependency ordering via topological sort, then calls OnInit() on each plugin. On failure, already-initialized plugins are not rolled back (caller should ShutdownAll + UnloadAll).
