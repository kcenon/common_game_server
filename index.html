Overview

common_game_server is a production-ready C++20 game server framework combining proven patterns from multiple game server implementations. It is the Tier 3 application layer of the kcenon ecosystem.

Key Features

Feature Description
Entity-Component System Data-oriented design with parallel execution
Plugin Architecture Hot-reloadable plugins with dependency resolution
Microservices 5 horizontally scalable services (Auth, Gateway, Game, Lobby, DBProxy)
Foundation Adapters 7 thin wrappers over kcenon foundation systems
Game Logic Systems Object, Combat, World, AI, Quest, Inventory
Security JWT RS256, TLS 1.3, input validation, SQL parameterization
Reliability WAL + snapshots, circuit breaker, graceful shutdown
Cloud-Native Kubernetes manifests, Prometheus, Grafana

Layered Architecture

Layer 6: PLUGIN LAYER (MMORPG, Battle Royale, RTS, Custom)
Layer 5: GAME LOGIC LAYER (Object, Combat, World, AI, Quest, Inventory)
Layer 4: CORE ECS LAYER (Entity, Component, Scheduler, Query)
Layer 3: SERVICE LAYER (Auth, Gateway, Game, Lobby, DBProxy)
Layer 2: FOUNDATION ADAPTERS (Result<T,E>, no exceptions)
Layer 1: KCENON FOUNDATIONS (common, thread, logger, network, db, container, monitoring)

Detailed architecture: see ARCHITECTURE.md

Quick Start

#include <cgs/cgs.hpp>
int main() {
using namespace cgs::foundation;
using namespace cgs::ecs;
// Create the foundation facade
auto foundation = GameFoundation::create_default();
foundation.logger().info("server starting");
// Set up an entity manager
EntityManager em;
auto e = em.create();
em.add<Position>(e, {0.0f, 0.0f});
// Schedule and tick systems
SystemScheduler sched;
sched.add<MovementSystem>();
sched.tick(em, 0.05f); // 50 ms (20 Hz)
return 0;
}
